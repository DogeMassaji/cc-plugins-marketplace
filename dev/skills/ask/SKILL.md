# Ask — 纯问答模式

## 概述

**只读问答模式。** 仅用已有知识或读现有文件回答用户问题。不写任何内容。

## 何时用

- 显式 `/ask` 调用

## 行为

### 允许

- 读文件（Read）
- 搜索文件（grep、find）
- 解释代码逻辑
- 分析架构
- 提供建议
- 回答技术问题
- 讨论方案

### 禁止

- 写或修改文件
- 创建新文件
- 运行可能改变状态的命令（git commit、npm publish 等）
- 生成代码（建议代码段可以，但不写入）
- 执行部署操作
- 运行测试（仅当只读时例外，如 `go build ./...` 编译检查，但建议跳过）

## 流程

### 1. 理解问题

理解用户问题，确认范围。

### 2. 只读探索

用以下方式收集信息：
- Read 现有文件
- 搜索文件系统
- 分析 git 历史
- 提供技术解释

### 3. 回答

- 清晰回答用户问题
- 引文件路径 + 行号
- 不确定时诚实说明

### 4. 拒绝写入

用户误提写入请求时：
> ⚠️ Ask 模式只读。如需写代码，不加 `/ask` 直接说就行。

## 规则

1. **不写任何东西**——文件、提交、产物都不写
2. **不生成代码文件**——代码片段可展示但不写入
3. **不运行 destructive 命令**——git push、deploy、publish 等禁止
4. **可以读**——Read、grep、git log 都行
5. **诚实**——不知道就说不知道

## 验证

- [ ] 未修改任何文件
- [ ] 未运行写操作
- [ ] 回答了用户问题
